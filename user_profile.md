# [개인 정보] 홍길동
## 지원 직무
- 백엔드 개발자

## 주요 스펙
- 학력/전공: 컴퓨터공학 학사
- 자격증: 정보처리기사, SQLD

## 핵심 경험 (원문)
1. 배경 (Situation)
학부 시절, 대학교 수강신청 기간의 고질적인 서버 지연 문제를 해결하고자 '교내 커뮤니티 수강신청 시뮬레이터' 프로젝트를 진행했습니다. 약 2,000명의 동시 접속자가 몰리는 상황을 가정하여 백엔드 설계를 담당했습니다.

2. 문제 (Problem)
초기 테스트 결과, 특정 인기 강의에 대한 조회 요청이 몰릴 때 응답 시간이 5초 이상 소요되는 병목 현상이 발생했습니다. 단순한 서버 증설보다는 백엔드 로직과 데이터베이스 구조 내에서 근본적인 원인을 찾아야 했습니다.

3. 해결 과정 (Action)
데이터베이스 정규화 및 인덱스 설계: SQLD 자격 취득 과정에서 익힌 지식을 바탕으로, 실행 계획(Execution Plan)을 분석했습니다. Full Table Scan이 발생하는 지점을 찾아 복합 인덱스(Composite Index)를 설정하여 조회 속도를 개선했습니다.

동시성 제어 처리: 여러 사용자가 동시에 수강 신청 버튼을 눌렀을 때 발생하는 'Race Condition'을 방지하기 위해 데이터베이스의 낙관적 락(Optimistic Lock)을 적용하여 정합성을 유지했습니다.

코드 리팩토링: 반복되는 DB 접근을 줄이기 위해 불필요한 쿼리를 통합하고, 자주 변하지 않는 강의 목록 데이터는 캐싱 처리하여 서버 부하를 40% 이상 경감시켰습니다.

4. 결과 (Result)
이러한 최적화를 통해 평균 응답 속도를 0.5초 이내로 단축시켰으며, 실제 시뮬레이션 환경에서 대규모 트래픽을 안정적으로 처리하는 데 성공했습니다.
