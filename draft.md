# 삼성전자 DX부문 SW개발(백엔드) 자기소개서 초안

## [주제: 데이터 기반의 집요함으로 응답 속도 10배를 개선하다]

**[Situation & Task: 2,000명의 동시 접속, 5초의 병목을 마주하다]**
학부 시절, 수강신청 기간의 고질적인 서버 지연 문제를 해결하기 위해 '교내 커뮤니티 수강신청 시뮬레이터' 프로젝트를 진행했습니다. 백엔드 설계를 담당하며 약 2,000명의 동시 접속자가 몰리는 극한의 상황을 가정하여 테스트를 진행했습니다. 초기 테스트 결과, 특정 인기 강의 조회 시 응답 시간이 5초 이상 소요되는 심각한 병목 현상이 발견되었습니다. 저는 단순한 서버 증설이 아닌, 코드와 아키텍처 수준에서의 근본적인 최적화가 필요하다고 판단했습니다.

**[Action 1: 기술적 무기를 활용한 아키텍처 최적화]**
먼저, 성능 개선을 위해 **Java 17**과 **Spring Boot 3.x** 환경에서 기술적 최적화를 단행했습니다. SQLD 자격증 취득 과정에서 익힌 지식을 바탕으로 실행 계획(Execution Plan)을 분석하여 Full Table Scan이 발생하는 지점에 복합 인덱스(Composite Index)를 설정했습니다. 또한, 서버 확장성을 고려하여 **Redis**를 도입해 자주 변하지 않는 강의 목록 데이터를 캐싱함으로써 반복적인 데이터베이스 접근을 줄였습니다. 이를 통해 서버 부하를 40% 이상 경감시켰습니다.

**[Action 2: 데이터 기반의 소통으로 팀의 합의를 이끌다]**
데이터 정합성을 유지하기 위한 '낙관적 락(Optimistic Lock)' 도입 과정에서 팀원들과의 의견 차이가 있었습니다. 성능 저하를 우려하는 팀원들을 설득하기 위해, 저는 주관적인 주장 대신 **JMeter를 활용한 벤치마킹 데이터**를 제시했습니다. 충돌 빈도가 낮은 환경에서는 낙관적 락이 비관적 락보다 자원 점유 시간이 짧아 전체 처리량(Throughput) 면에서 유리함을 수치로 증명했습니다. 결과적으로 팀원들의 기술적 합의를 이끌어내며 성공적으로 기능을 구현할 수 있었습니다.

**[Result: 0.5초의 혁신, 삼성전자의 미래를 잇다]**
이러한 집요한 최적화 결과, 평균 응답 속도를 5초에서 **0.5초 이내로 단축**시키는 10배의 성능 향상을 달성했습니다. 이 과정에서 배운 '데이터 기반의 의사결정'과 '아키텍처 최적화 역량'은 대규모 트래픽을 처리하는 삼성전자의 백엔드 시스템 안정성을 확보하는 데 기여할 것입니다. 기술적 한계를 데이터로 극복하며, 전 세계 사용자가 체감하는 최고의 품질을 실현하는 개발자가 되겠습니다.
